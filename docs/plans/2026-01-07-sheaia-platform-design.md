# SHEAIA Platform Design

> **AI-Powered Enterprise Insights Platform**
> 
> "Talk to your enterprise data" - Natural language access to CRM, ERP, MES, and documents

---

## Executive Summary

SHEAIA is an on-premise AI appliance that provides enterprise customers with natural language access to their business data. Built on LangGraph multi-agent architecture and self-hosted LLMs, it connects to enterprise systems (CRM, ERP, MES), documents, and databases to deliver unified insights through a conversational interface.

### Key Value Propositions

- **100% Local & Secure** - All data stays on-premise, air-gapped capable
- **AI-Native Data Access** - Ask questions in natural language, get answers with citations
- **Cross-System Intelligence** - Correlate data across CRM, ERP, MES seamlessly
- **Self-Learning** - AI agents continuously discover and index data sources

---

## Target Customers

| Segment | Description | Key Needs |
|---------|-------------|-----------|
| **SMEs** | Small-to-medium enterprises (10-200 employees) | Affordable AI insights without cloud dependency |
| **Departments** | Single department in larger enterprise | Focused analytics without IT overhead |
| **Manufacturing** | Manufacturing companies with MES | Production data correlation with business systems |

---

## Core Capabilities

| Priority | Capability | Description |
|----------|------------|-------------|
| **P0** | Natural Language Queries | Ask questions about data in plain language |
| **P0** | Document Intelligence | RAG over contracts, reports, manuals |
| **P0** | Cross-System Correlation | Unified view across CRM/ERP/MES |
| **P1** | Automated Reporting | Generate periodic reports |
| **P1** | Anomaly Detection | Alert on unusual patterns |
| **P2** | Production Analytics | OEE, downtime analysis (via MES) |

### Agent Specifications

| Agent | Trigger | State Managed | Key Tools |
|-------|---------|---------------|-----------|
| **Coordinator** | Every user request | Conversation context, task queue | `delegate_task`, `synthesize_results` |
| **Query Agent** | "Find/Show/Compare..." | Query history, cached results | `sql_execute`, `data_transform`, `visualize` |
| **Document Agent** | "What does X say about..." | Retrieved chunks, citations | `vector_search`, `extract_entities`, `summarize` |
| **Connector Agent** | Setup/sync requests | Connection state, credentials | `test_connection`, `fetch_schema`, `sync_data` |
| **Schema Agent** | Background + on-demand | Discovered schema graph | `explore_tables`, `infer_relationships`, `merge_schema` |
| **Indexer Agent** | Background continuous | Index state, processing queue | `chunk_document`, `embed_text`, `update_index` |
| **Report Agent** | Scheduled + on-demand | Report templates, schedules | `generate_report`, `export_pdf`, `send_notification` |

### Key Libraries from LangGraph Ecosystem

| Library | Purpose in SHEAIA |
|---------|-------------------|
| `langgraph-supervisor` | Coordinator agent orchestration |
| `langmem` | Persistent agent memory across sessions |
| `langgraph-bigtool` | Managing many connector tools efficiently |
| `langchain-mcp-adapters` | Future MCP tool integrations |

### Optimization Techniques

| Technique | Description | Benefit |
|-----------|-------------|---------|
| **Quantization (Q4/Q5)** | 4-5 bit weight quantization | Fit 32B model in ~18GB |
| **KV Cache Optimization** | Paged attention, sliding window | Handle long contexts |
| **Speculative Decoding** | Small draft model + verify | 2-3x faster generation |
| **Continuous Batching** | Dynamic request batching | Better GPU utilization |
| **NPU Offload** | Embeddings on NPU | Free GPU for LLM |

## Implementation Roadmap

### Phase 1: Foundation (8 weeks)

**Goal:** Basic chat interface with local LLM, file import, vector search

- Week 1-2: Development environment setup (NVIDIA server)
- Week 3-4: Core infrastructure (FastAPI, LLM service, embeddings)
- Week 5-6: Basic features (chat, file upload, document RAG)
- Week 7-8: Web UI v1 (chat interface, file upload)

**Deliverables:** Chat API, File Import, Document RAG, Basic Web UI

---

### Phase 2: Core Intelligence (10 weeks)

**Goal:** Multi-agent system, schema discovery, cross-source queries

- Week 1-3: Agent framework (LangGraph, Coordinator, Query, Document agents)
- Week 4-5: Schema discovery (introspection, semantic labeling, relationship inference)
- Week 6-7: Cross-source queries (multi-source planning, join execution)
- Week 8-9: Generic connectors (ODBC, REST, file watcher)
- Week 10: Integration & testing

**Deliverables:** Agent System, Schema Discovery, Cross-Source Query, Generic Connectors

---

### Phase 3: Enterprise Features (8 weeks)

**Goal:** Pre-built connectors, messaging integration, reports

- Week 1-3: Pre-built connectors (Kingdee, Yonyou, 2x MES)
- Week 4-5: Messaging integration (WeChat Work, Feishu)
- Week 6-7: Reporting system (templates, scheduling, export)
- Week 8: Admin & security (users, RBAC, audit log)

**Deliverables:** Pre-built Connectors, WeChat/Feishu Bots, Report System, Admin UI

---

### Phase 4: Production (6 weeks)

**Goal:** AMD platform support, appliance packaging, customer pilot

- Week 1-2: AMD platform port (ROCm, NPU, ONNX Runtime)
- Week 3-4: Appliance build (OS image, Docker, auto-start)
- Week 5: Quality assurance (security audit, stress testing)
- Week 6: Customer pilot

**Deliverables:** AMD Support, Appliance Image, Documentation, Pilot Deployment
